[BST 1부] 이진탐색트리

정의

이진탐색트리는 이진탐색의 장점인 탐색과 연결리스트의 장점인 삽입, 삭제 연산을 결합한 자료구조이다.

이진탐색트리를 논하기 이전에 이진탐색과 연결리스트에 대해 간단히 언급하고자 한다.

이진탐색

탐색은 순차탐색과 이진탐색으로 나뉜다.

  • 순차탐색 (Sequential Search)
    순차탐색의 경우 첫번째 위치부터 시작하여 위치를 이동해나가면서 탐색 연산을 수행한다. 코드로 구현하기는 간단하나, 탐색 연산을 완료하기까지 $O(N)$만큼의 시간복잡도가 요구된다. 따라서, 찾고자 하는 데이터가 후위에 위치하거나 데이터의 양이 증가할수록 연산의 수행속도가 느려지는 단점이 존재한다. 
    bool sequential_search(int target, vector<int> &dataset) {
  bool result = false;

  for(auto data : dataset){
      result = data == target;
      if(result) break;
  }

  return result;
}
  • 이진탐색 (Binary Search)
    이러한 순차탐색의 수행속도를 획기적으로 감소시키기 위해 이진탐색은 아래와 같은 순서로 탐색을 수행한다.
    1. 첫번째 위치와 마지막 위치의 중간부터 시작하여 탐색을 수행한다.
    2. 중간위치에 존재하는 데이터가 찾고자하는 데이터보다 작으면 왼쪽의 집합으로, 크다면 오른쪽의 집합으로 이동하여 이동한 부분 집합에서 중간에 위치한 값과 찾고자 하는 값의 탐색 연산을 반복한다.

이를 통해 탐색 연산을 위한 시간복잡도는 $O(logN)$이 되므로 순차탐색에 비했을때 월등히 빠른 속도로 연산을 수행하는 것이 가능하다.

bool binary_search(int target, vector<int> &dataset) {
    int start = 0; 
    int end = dataset.size() - 1;
    bool result = false;

    while(start > end){
        int mid = (start + end) / 2;
        result = dataset[mid] == target;

        if(result) break;

        if(dataset[mid] > target) end = mid - 1;
        else start = mid + 1;   
    }

    return result;
}

연결리스트

연결리스트는 컴퓨터 관련학과에서 자료구조 수업을 듣는다면 서두에 언급되는 기초적 선형구조 중 하나다.

연결리스트의 각 노드들은 연결된 노드의 주소값을 가지고 있기에, 삽입 및 삭제 연산 수행 시 연산이 발생하는 위치와 인접한 노드들이 가진 주소값을 변경해주기만 하면 된다.
따라서, 시간복잡도를 놓고 본다면 $O(1)$만에 삽입 및 삭제 연산을 수행할 수 있다.

이진탐색트리

이진탐색이 탐색에 있어서는 $O(logN)$의 시간복잡도를 가진다는 장점이 있으나, 삽입과 삭제 연산에 있어서는

연결리스트의 경우 삽입과 삭제 연산에 있어서는 $O(1)$의 시간복잡도를 가지나, 탐색에 있어서는 순차탐색을 수행하므로 $O(N)$의 시간복잡도를 가진다는 단점이 있다.

이진탐색트리는 서두에도 언급했듯이 이진탐색의 장점을 가져가고 이진탐색의 단점을 보완하기 위해 연결리스트의 장점을 혼합한 비선형 자료구조이다.
